# CS-300
## Module Eight Journal Reflection (CS 300)

In CS 300, the main problem I worked on was how to store, search, and sort course information efficiently. The projects focused on taking course data (including prerequisites), organizing it in a useful way, and making it easy to either print the full list in alphabetical order or look up one specific course. Overall, the goal was to practice using data structures and understand how different choices affect performance and usability.

My approach was to start by thinking about what the program needed to do most: load data, find items quickly, and output results in a clean format. That’s where data structures really matter, because the structure you choose can make the difference between a program that stays fast and organized versus one that gets slow or messy as the amount of data grows. Comparing different options helped me see the tradeoffs between speed, memory use, and how complicated the code becomes.

The biggest roadblocks I ran into were mostly related to handling input correctly and getting the program output to match the requirements. When something wasn’t working, I fixed it by testing one part at a time instead of trying to solve everything at once. I would load the data first, confirm it looked right, then move on to printing, then searching. Breaking it into smaller steps made debugging a lot less overwhelming and helped me pinpoint exactly where a problem was coming from.

This course expanded the way I think about designing software because it pushed me to plan structure and flow before jumping into code. Instead of writing everything in one big block, I focused on building a clear menu system and separating the program into logical parts. It also helped me get better at thinking ahead—like choosing a structure that supports the program features now, but could also be expanded later without rewriting everything.

I also improved the way I write code so it’s easier to read and maintain. I leaned more into clean function design, meaningful naming, and keeping responsibilities separate (loading data, searching, printing output). That makes the program easier to debug and easier for someone else to understand. It also makes it more adaptable, because changes can be made in one area without breaking the entire program.
